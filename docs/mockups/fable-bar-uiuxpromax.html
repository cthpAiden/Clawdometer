P3 — Rail cluster, shared reset caption · 200×112, 120×92 (no growth)
§6 whitespace-balance "use whitespace to group related items" · §5 content-priority "show core content first"
Hero number keeps the left, all three rails cluster on the right as one visual group. Grouping is the point: 7D and Fable reset at the same instant, so one caption serves both and you stop repeating "resets Thu 9:59am" twice. Fits in the current 112px with no growth. Weakness: 3px rails at ~90px of track — readable as "roughly where am I", not as a precise gauge. The numbers carry the precision.

P4 — Segmented meters · 200×112, 120×92 (no growth)
chart.csv a11y-fallback "color ranges labeled with threshold text" · §10 pattern-texture "supplement color with patterns so data is distinguishable without color"
Ten discrete segments instead of a continuous fill. At HUD scale this is the honest move: a 120px continuous bar implies a precision it does not have, while "6 of 10 lit" is countable at a glance and survives both a bad monitor and a colourblind reader, because the segment boundary is a shape, not a hue. Last two segments render amber, so the danger zone has texture as well as colour. The tradeoff is a slightly technical, equalizer-ish look, and you lose the sub-10% resolution.

P5 — Adaptive hero metric · 200×112, 120×92 (no growth)
§5 content-priority · §4 primary-action "each screen should have only one primary CTA; secondary visually subordinate"
The card auto-promotes whichever window is closest to cutting you off into the hero number, and demotes the other two to muted rails. Right now the 26px number is hard-wired to the 5-hour session, which is a guess about what matters — with three windows that guess is wrong two-thirds of the time. The chip tells you which window you're looking at. Second card shows Fable promoted. Real risk: the hero number changes meaning underneath you, so a glance-and-go read can be misread. Only works if the chip is unmissable.
