Drops into your statusline. Doesn't fight it.

If you already had a statusline command configured, Clawdometer chains it instead of replacing it.

~/.claude/settings.json
-"statusLine": "my-custom-script.ps1"
+"statusLine": "clawdometer wrap"
uninstall restores the original line exactly, from a full backup taken before install ever touched it.
powershell
PS> clawdometer install
  backing up settings.json to ~/.clawdometer/backups/
  ok chained your existing statusline, 2s timeout
PS> clawdometer uninstall
  ok restored original statusLine command
